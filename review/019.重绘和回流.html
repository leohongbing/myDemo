浏览器渲染过程

将html解析为dom tree，style解析为style tree

dom tree 和style tree 生成render tree

回流：render tree大小位置发生改变，引起浏览器重新渲染页面

重绘：render ttee的颜色、透明度等发生改变

避免回流

1. 批量修改dom样式

2. 避免反复读取同一尺寸属性，用值缓存

3. 复杂动画开启定位

4. 开启css3硬件加速，GPU渲染页面

常见的触发硬件加速(合成层)的css属性：

transform/ opacity/ filters/ Will-change transform: translateZ(0)

硬件加速的坑

如果你为太多元素使用css3硬件加速，会导致内存占用较大，会有性能问题。

在GPU渲染字体会导致抗锯齿无效。这是因为GPU和CPU的算法不同。因此如果你不在动画结束的时候关闭硬件加速，会产生字体模糊
